## Architecture

### Overview
TeamClaw v2 is an AI Agent management platform serving as an enhanced frontend for **OpenClaw Gateway**. Built with Next.js 14 App Router + TypeScript strict mode. It uses a **dual data source architecture**: local SQLite for project data (tasks, docs, members) and WebSocket for Gateway data (agents, sessions, cron, skills).

### Data Flow (4 paths)

1. **User → SQLite**: React component → Zustand Store → `lib/data-service.ts` (fetch) → `app/api/*` Route Handlers → Drizzle ORM → SQLite (`data/teamclaw.db`)
2. **User → Gateway**: React component → Zustand Store → `lib/gateway-client.ts` (WebSocket, Protocol v3) → OpenClaw Gateway
3. **Real-time push**: API write triggers `eventBus.emit()` (`lib/event-bus.ts`) → SSE stream at `/api/sse` → `DataProvider` component auto-refreshes stores
4. **AI MCP commands**: AI Agent → `/api/mcp` (or `/api/mcp/external` with Bearer token) → `core/mcp/` executor → DB write + SSE broadcast → frontend refresh

### Database Layer (`db/`)

- **`db/schema.ts`**: 21 Drizzle ORM tables. All primary keys are Base58 short IDs (not UUID), generated by API layer. Timestamps use `integer` with `mode: 'timestamp'`. JSON fields use `text` with `mode: 'json'` + `$type<T>()`. Every table exports both `Select` and `Insert` types.
- **`db/index.ts`**: Singleton connection via `globalThis` (HMR safe). Enables WAL mode, foreign keys, 64MB cache, 5s busy timeout. Includes auto-migration logic that detects missing tables/columns and adds them incrementally—never manually edit `data/teamclaw.db`.

### Gateway Client (`lib/gateway-client.ts`)

WebSocket client implementing OpenClaw Protocol v3. Handshake: challenge → connect (token + role `operator`) → `hello-ok`. Request/response model with 30s timeout. Auto-reconnect with exponential backoff (1s–60s, max 10 retries). Global singleton via `createGatewayClient()`/`getGatewayClient()`. Covers: snapshot, agents (CRUD + files), sessions, cron, skills, config, chat.

### API Routes (`app/api/`)

21 API modules, all RESTful. Key conventions:
- Collection routes: `route.ts` → GET (list) + POST (create)
- Resource routes: `[id]/route.ts` → GET + PUT + DELETE
- Dynamic params: `{ params }: { params: Promise<{ id: string }> }` with `await params`
- PUT/DELETE must check resource existence (404 if missing)
- Sensitive fields (e.g. `openclawApiToken`) must pass through `lib/sanitize.ts` before response
- PUT uses `allowedFields` whitelist; cascading deletes use `db.transaction()`
- Error format: `{ error: string }` with appropriate HTTP status

### MCP System (`core/mcp/`)

Three-layer architecture for AI Agent tool calls:
- **`types.ts`**: 27 `ActionInstruction` types + `ExecutionResult` + `PendingQuestion`
- **`definitions.ts`**: 37 MCP tool JSON Schema definitions (name, description, parameters)
- **`executor.ts`**: Switch-case dispatcher; calls `/api/mcp` internally, refreshes Zustand stores on success
- **`app/api/mcp/handlers/`**: 9 domain handlers (task, document, member, project, schedule, status, delivery, template, sop)

To add a new MCP tool: (1) register definition in `definitions.ts`, (2) add handler in `handlers/`, (3) add switch-case in `executor.ts`.

### Store Layer (`store/`)

16 Zustand stores organized by domain. All re-exported from `store/index.ts`. `useDataInitializer()` hook loads all data in parallel via `Promise.allSettled` on app startup.

Critical rules:
- `updateAsync`/`createAsync` must update local state with **API response data**, not the request body
- `deleteAsync` must **await** API success before removing from local state
- Success operations must `set({ error: null })`

### Frontend Components (`components/`)

- **Layout**: `AppShell` (main shell), `Header`, `Sidebar`, `ThemeProvider`
- **Data**: `DataProvider` connects SSE stream and auto-refreshes stores on server events
- **UI**: `components/ui/` has 15 shadcn/ui primitives—must use these, never re-implement
- **Chat**: `chat/ChatPanel` (orchestrator ~670 lines), `ChatInputArea`, `ChatMessageList`, `ChatSessionList`, `ChatDrawer`, `ChatFab`; reusable hooks: `useChatStream`, `useAutoScroll`
- **SOP**: `sop/SOPProgressBar` (compact/expanded progress), `SOPTemplateEditor` (drag-sort stages), `SOPDebugPanel`
- **OpenClaw**: `openclaw/WorkspaceCard`, `WorkspaceForm`

### Middleware (`middleware.ts`)

Applies only to `/api/*`. Adds security headers, CORS (localhost in dev, `ALLOWED_ORIGINS` in prod), CSRF protection for mutating requests (Origin/Referer check), 1MB body limit, and `X-Request-Id` UUID per request.

### Internationalization (`lib/i18n.ts`)

All user-visible text must use `react-i18next` `t()` function—no hardcoded strings. Single `translation` namespace with nested keys by module. Dual language: `en` + `zh`. Language detection: localStorage → browser → fallback `en`. Translated constants must be defined inside components after `useTranslation()`.

### Key Patterns

- **Path alias**: `@/*` maps to project root; no cross-layer relative imports
- **Input debounce**: All edit inputs must debounce 500ms via `useRef<setTimeout>`
- **Enter/Blur guard**: Inline edits use `submittedByEnterRef` to prevent double-submit
- **Derived lists**: Always `useMemo` for filter/map/sort operations
- **useEffect deps**: Must be specific fields (`[task?.id]`), never whole objects (`[task]`)
- **Dark mode**: `darkMode: 'class'` via `ThemeProvider`; all components must support both themes
- **CSS**: Tailwind only, no `<style jsx>`; theme via CSS variables (`--background`, `--text-primary`, etc.)
- **Icons**: `lucide-react` exclusively; conditional classNames via `clsx()`
- **Comments**: Write code comments in Chinese
